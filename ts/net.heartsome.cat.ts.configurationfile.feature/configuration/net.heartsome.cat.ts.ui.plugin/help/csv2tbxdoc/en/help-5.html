Glossary

  • CSV (comma separated values)

    Plain text file where data is listed in columns, each value being separated by a comma. Each new line represents a new set of data.

  • TBX

    TBX is an open XML-based standard format for terminological data. This standard provides a number of benefits so long as TBX files can be imported into and exported from most software packages that include a terminological database. Visit http://www.lisa.org/tbx for more information.

  • Column Separator

    Character used to separate data columns . Usually a comma.

  • Text Delimiter

    Character used to delimiter the content of a data column. Double quotes are frequently used as delimiters. Delimiters are required whenever column text contains the character used as Column Separator.

  • Character Set

    The set of numeric codes used by a computer system to represent the characters (letters, numbers, punctuation, etc.) of a particular country or place. Unix based operating systems (Linux and Mac OS) use UTF-8 as default character set. Microsoft Windows uses propietary encodings known as Windows-XXX. Many European countries use an extension of ASCII known as ISO-8859-1 (ISO Latin-1).

  • Main Language

    Language used in the notes and properties of all concepts.

  • XCS Template

    The XCS Template is the XML file that contains the list of attributes that can be specified in a term. Strictly speaking, the template is a Data Constraint Specification.
