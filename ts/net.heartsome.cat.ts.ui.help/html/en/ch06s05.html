5. Export TM as TBX

Just a TMs can be exported and imported as TMX files, termbase data can be exchanged across different CAT tools by exporting and importing TBX files. Here is how:

  1. Click Databases > Export TBX,

    Figure 140. Export Termbases as TBX Files


  2. In the Export Termbases as TBX Files dialog box, the following settings need to be configured:

    Figure 141. Export termbase as TBX


    • Termbase

      • Add

        Click the button, then select the type of TM to add.

        If you choose file-based TM, the Open dialog will pop pu. Then browse to and select .hstm file to be exported.

        Figure 142. Select file-based termbase


        If you select server-based termbase, Termbase Management dialog box will pop up. From there, select the termbase to be exported and click OK.

        Figure 143. Select server-based termbase


      • Remove

        Remove a termbase from the list of "termbases to be exported".

      • Export Languages

        When a termbase from the list is selected, the languages of the termbase will be shown here. You must select at least two languages to export the termbase. Select all by default.

      • Source Language

        Specify the source language for the TBX export.

    • Filter Rules

      • Add

        When the selected filter rule is “N/A”, click Edit to add a new filter rule. A filter rule includes the following information:

        Figure 144. Export TBX: Filter rules


        • Name

        • Type of criteria combination

          Meet all of the following conditions, meet one of the following conditions

        • Criteria

          • Term

      • Edit

        When you select a filter rule that has already been saved, you can click the Edit button to edit the rule.

      • Delete

        Delete the selected filter rule.

    • Custom Coding

      Specify the encoding used for TMX files. The default coding is “UTF-8” and usually does not need to be changed.

    • TBX save path
